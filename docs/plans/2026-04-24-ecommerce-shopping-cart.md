# E-commerce Shopping Cart — Implementation Plan

> **For Claude:** Use `executing-plans` to implement this plan task-by-task.

**Goal:** Build a full-stack single-page e-commerce app with product browsing, shopping cart, JWT auth, live search, and admin dashboard — fulfilling all 32516 Assignment 2 rubric criteria.

### Task 1 — Database Schema and Seed Data
**Owner:** Qiushi Huang
**Files:** `database/schema.sql`, `database/seed.sql`

**schema.sql — 4 tables:**
- `users`: id, username (UNIQUE), email (UNIQUE), password_hash, role ENUM('user','admin'), created_at

**seed.sql — initial data:**
- 1 admin user (placeholder password hash — replaced in Task 12 with real bcrypt hash)
- 5 sample products with placeholder image URLs

### Task 3 — User Model, Schemas, and JWT Auth
**Owner:** Xuyu Zhang
**Files:** `backend/models/user.py`, `backend/schemas/user.py`, `backend/auth/jwt.py`, `backend/routers/auth.py`

**models/user.py:** SQLAlchemy ORM mapping for `users` table — all 6 columns.

**schemas/user.py — 5 Pydantic classes:**
- `UserRegister`: username (min 3, max 50), email (EmailStr), password (min 6)
- `UserLogin`: email, password
- `UserUpdate`: optional username, optional email (for `PUT /users/me`)
- `UserOut`: id, username, email, role — returned to frontend (never includes password)
- `Token`: access_token, token_type="bearer", user (UserOut)

**auth/jwt.py — 5 functions:**
- `hash_password(password)` → bcrypt hash
- `verify_password(plain, hashed)` → bool
- `create_token(data)` → JWT string with expiry from env
- `get_current_user(credentials, db)` → decodes Bearer token, returns User or raises 401
- `require_admin(current_user)` → checks role=="admin" or raises 403

**routers/auth.py — 2 endpoints:**
- `POST /auth/register` (201): hash password → save user → `db.flush()` to get user.id before commit → create empty ShoppingCart for the new user → `db.commit()` → return Token
- `POST /auth/login`: verify email + password → return Token

> Note on `db.flush()`: called before `db.commit()` so the new user's `id` is available in the same transaction to create the linked shopping cart.

**Verification:** Test both endpoints in Swagger → expect JWT token in response.

**Commit:** `feat: implement JWT register and login with bcrypt`

**Owner:** Qiushi Huang

**models/product.py:** ORM mapping for `products` table — all 7 columns.

**schemas/product.py — 3 Pydantic classes:**
- `ProductCreate`: name, description (optional), price, stock (default 0), image_url (optional)

**routers/products.py — 5 endpoints:**

- `POST /products` (admin only): create product
- `PUT /products/{id}` (admin only): partial update using `exclude_unset=True`
- `DELETE /products/{id}` (admin only): delete or 404

**Verification:** Test all 5 endpoints in Swagger; confirm non-admin token is rejected on POST/PUT/DELETE.

### Task 4b — Cart Model and Routes
**Owner:** Xuyu Zhang
**Files:** `backend/models/cart.py`, `backend/schemas/cart.py`, `backend/routers/cart.py`

**models/cart.py — 2 ORM classes:**
- `ShoppingCart`: id, user_id (FK, unique), created_at; has `items` relationship
- `CartItem`: id, cart_id (FK), product_id (FK), quantity; has `product` relationship; UNIQUE(cart_id, product_id)

**schemas/cart.py — 5 Pydantic classes:**
- `CartItemAdd`: product_id, quantity (default 1)
- `CartItemUpdate`: quantity (int)
- `CartItemOut`: id, product (ProductOut), quantity
- `CartOut`: id, items (list of CartItemOut)
- `UserCartOut`: user_id, username, cart (CartOut) — used by admin endpoint

**routers/cart.py — 5 endpoints:**
- `GET /cart/me` (login required): query `shopping_cart` by `user_id`; if None (e.g. admin seeded without cart), raise `HTTPException(404, "Cart not found")` — do not return None to the frontend
- `POST /cart/items` (login required): add item; if product already in cart, increment quantity instead of creating duplicate
- `PUT /cart/items/{id}` (login required): update quantity; if quantity ≤ 0, delete the item automatically
- `DELETE /cart/items/{id}` (login required): remove item
- `GET /cart/all` (admin only): return all users' carts using `joinedload` to avoid N+1 queries

**Verification:** Add item, view cart, update qty, remove item — all via Swagger with a user token.

**Commit:** `feat: add shopping cart CRUD routes`

**Setup:**

**main.jsx:** Wrap the entire app in `<BrowserRouter>` and `<AuthProvider>` so every component can access routing and auth state.

**App.jsx — routing table:**
- `/` → `HomePage` (public)
- `/login` → `LoginPage` (public)
- `/register` → `RegisterPage` (public)
- `/cart` → `CartPage` (requires login — `PrivateRoute` wrapper)

**PrivateRoute:** If `user` is null → redirect to `/login`. Otherwise render children.

**AdminRoute:** If `user.role !== 'admin'` → redirect to `/`. Otherwise render children.

**Catch-all route:** Add `<Route path="*" element={<Navigate to="/" replace />} />` as the last route — prevents blank page on unknown URLs.

- Responsive breakpoints: minimum support for 768px (tablet) and 1280px (desktop); product grid uses `grid-template-columns: repeat(auto-fill, minmax(220px, 1fr))` for automatic responsiveness

**Owner:** Xuyu Zhang
**Files:** `frontend/src/context/AuthContext.jsx`, `frontend/src/services/api.js`, `frontend/src/services/authService.js`, `frontend/src/services/cartService.js`

**AuthContext.jsx:**
- Initialise `user` state from `localStorage` (persist login across page refresh)
- `login(userData, token)`: save token + user to `localStorage`, update state
- `logout()`: clear `localStorage`, reset state
- Export `useAuth()` hook for convenient access in any component

**api.js — centralised fetch wrapper:**
- `request(method, path, body)`: reads token from `localStorage`, sets `Authorization: Bearer` header automatically
- Handles 204 No Content responses (return null)
- Throws `Error` with `data.detail` message on non-OK responses — ensures errors surface cleanly, no silent failures
- Exports `api.get`, `api.post`, `api.put`, `api.delete`
- Base URL from `import.meta.env.VITE_API_URL` — never hardcode

**authService.js:** `register(data)`, `login(data)` — thin wrappers over `api.post`

**cartService.js:** `getMyCart()`, `addToCart(productId, qty)`, `updateCartItem(itemId, qty)`, `removeCartItem(itemId)`, `getAllCarts()`

### Task 8 — HomePage with Live Search
**Owner:** Qiushi Huang
**Files:** `frontend/src/pages/HomePage.jsx`, `frontend/src/components/ProductCard.jsx`, `frontend/src/hooks/useSearch.js`

**useSearch.js (custom hook):**
- State: `query`, `products`, `loading`, `error`
- `useEffect` watches `query`: on change, sets a 300ms `setTimeout` before calling `getProducts(query)` (debounce prevents excessive API calls on every keystroke)
- Cleans up timer on re-render with `clearTimeout`
- On API failure: set friendly `error` message — never throw to the UI

**ProductCard.jsx:**
- Displays: product image, name, description, price, stock status (green "X in stock" / red "Out of stock")
- "Add to Cart" button: disabled when stock === 0
- If not logged in: show "Please login" message inline
- On add success: show "Added!" feedback for 2 seconds then clear
- On add failure: show error message inline

**HomePage.jsx:**
- Renders search input wired to `setQuery` from `useSearch`
- Renders `products.map(p => <ProductCard .../>)` in a flex-wrap grid
- Shows "Loading..." while `loading === true`
- Shows `error` message if fetch failed
- Shows "No products found." if results empty and not loading

**Verification:** Type in search box → product list updates without page reload; no submit button needed.

**Commit:** `feat: add HomePage with live search and ProductCard`
